Getting Started

Quick Start

Get NovaStar Engine running in under a minute. You'll need Node.js 18+ installed.

Install & Run

Clone the repo and start the dev server:

Terminal
# Clone and enter the project
git clone https://github.com/JustyyDev/NovaStar-Engine.git
cd NovaStar-Engine

# Install dependencies
npm install

# Start the web dev server
npm run dev

# Or launch as a desktop app (recommended)
npm run electron:dev

Choose a Template

When the editor opens, you'll see the Welcome Screen with project templates. Pick one to start with pre-configured entities, prefabs, and settings — or choose Blank Project for a clean slate.

💡

Tip: The 3D Platformer template is the best way to learn — it comes with a player, ground, collectible stars, and all the physics pre-configured.

Add Entities

Use the Quick Add palette at the bottom of the editor to drop entities into your scene. Click any item (Platform, Character, Star, Enemy, etc.) and it appears in the viewport.

Select entities in the Scene Hierarchy (left panel) and adjust position, rotation, and scale in the Inspector (right panel).

Press Play

Hit F5 or click the Play button to test your game. Use WASD to move, Space to jump, and Shift to dash. Press F5 again to stop and return to the editor.

New in v0.3

UI System

Build health bars, score displays, buttons, timers, inventory grids, and pause menus — all from the visual editor.

Reusable Templates

Prefab System

Prefabs let you create reusable entity templates. Define an entity once with all its components, then spawn instances throughout your scene. When you update the prefab, all instances update too.

Creating a Prefab

There are two ways to create prefabs:

From the editor: Right-click any entity in the Scene Hierarchy and select "Make Prefab". The entity becomes a prefab template in your library.

From code:

JavaScript
import { Prefab, PrefabLibrary } from './engine/ProjectManager.js';

// Create a prefab library
const library = new PrefabLibrary();

// Define a reusable enemy prefab
const slimePrefab = library.add(new Prefab({
  name: 'Slime Enemy',
  category: 'enemy',
  components: [
    { type: 'ToonMesh', meshType: 'enemy', color: '#dd4444' },
    { type: 'PhysicsBody', gravity: -25, tag: 'enemy' },
    { type: 'PatrolAI', speed: 2, range: 5 },
    { type: 'Health', maxHP: 3 },
  ],
  tags: ['enemy'],
}));

// Spawn an instance at a specific position
const enemy1 = slimePrefab.instantiate({
  position: { x: 5, y: 2, z: 0 }
});

// Spawn another instance elsewhere
const enemy2 = slimePrefab.instantiate({
  name: 'Boss Slime',
  position: { x: -10, y: 2, z: 3 },
  scale: { x: 2, y: 2, z: 2 }
});

Updating All Instances

When you modify a prefab, use applyToInstances to push changes to every instance in a scene:

JavaScript
// Update the prefab's health component
slimePrefab.components[3].maxHP = 5;

// Apply to all instances in the current scene
const count = library.applyToInstances(slimePrefab.id, currentScene);
console.log(`Updated ${count} instances`);
📌

Prefab instances in the hierarchy show a purple left border and a "(prefab)" label. Instance-specific overrides (like position) are preserved when applying prefab changes.

Level Design

Tilemap Editor

The tilemap editor lets you paint tile-based levels directly in the editor. It supports multiple layers, a configurable tileset, and automatic collision generation.

Using the Tilemap Editor

Switch to the Tilemap Editor tab in the bottom panel. Select a tile color from the palette on the left, then click and drag on the grid to paint tiles. The editor supports paint, fill, eyedropper, and rectangle tools.

Tilemap in Code

JavaScript
import { TilemapData } from './engine/ProjectManager.js';

// Create a 16x12 tilemap
const tilemap = new TilemapData({
  width: 16,
  height: 12,
  cellSize: 1,  // 1 world unit per cell
});

// Paint individual tiles (layerIndex, x, y, tileId)
tilemap.setTile(0, 5, 10, 1);  // Grass at (5,10)
tilemap.setTile(0, 6, 10, 3);  // Stone at (6,10)

// Fill a rectangular region
tilemap.fill(0, 0, 10, 15, 11, 1);  // Grass floor

// Generate physics colliders from solid tiles
const colliders = tilemap.generateColliders();
// Returns array of { position, halfExtents, isStatic, tag }
💡

Auto-collision: The tilemap automatically generates optimized AABB colliders from solid tiles using greedy meshing. No manual collision setup needed.

Sound Design

Audio File Loading

v0.3 adds support for loading external audio files alongside the built-in synthesized sounds. Import WAV, MP3, and OGG files for custom sound effects and music.

Importing Audio in the Editor

Open the Audio tab in the right panel. Click "Import Audio File" to add files. You'll see a waveform preview, format info, and playback controls for each imported sound.

Using Audio in Code

JavaScript
// Built-in synthesized sounds still work
engine.audio.play('jump');
engine.audio.play('collect');

// Load and play external audio files
const bgMusic = await engine.audio.loadFile('assets/bg_music.mp3');
engine.audio.playLoaded(bgMusic, {
  loop: true,
  volume: 0.6,
  category: 'music'
});

// Spatial audio: attach sound to an entity
engine.audio.playSpatial('waterfall.ogg', {
  position: entity.transform.position,
  maxDistance: 20,
  rolloff: 1.5
});

The built-in synthesized sounds (jump, collect, dash, bounce, hurt, powerup, explosion, etc.) are still available and require no file loading.

Game Interface

Basic UI System

Build in-game interfaces with the new UI system. Create health bars, score displays, buttons, timers, inventory grids, and menu screens — either visually in the editor or via code.

Adding UI in the Editor

Switch to the UI tab in the right panel. Click any UI element to add it to the scene. Position and configure it in the Inspector.

UI in Code

JavaScript
import { UIElement } from './engine/ProjectManager.js';

// Create a health bar
const healthBar = new UIElement({
  type: 'healthBar',
  anchor: { x: 0, y: 0 },     // top-left
  offset: { x: 20, y: 20 },
  size: { width: 200, height: 24 },
  properties: {
    maxValue: 100,
    currentValue: 100,
    barColor: '#4ee6a0',
    showLabel: true
  }
});

// Create a score display
const score = new UIElement({
  type: 'scoreDisplay',
  anchor: { x: 1, y: 0 },     // top-right
  offset: { x: -20, y: 20 },
  properties: {
    score: 0,
    prefix: 'Stars: ',
    color: '#ffe066'
  }
});

// Update UI at runtime
healthBar.properties.currentValue = 75;
score.properties.score += 1;
⚠️

Note: UI elements use screen-space positioning with anchors (0–1). Anchor {x:0, y:0} is top-left, {x:1, y:1} is bottom-right. The offset is in pixels from the anchor point.

Create Models

Built-in 3D Modeler

NovaStar has a built-in 3D model builder so you can create custom low-poly models right inside the editor. No external software needed. Build characters, props, buildings, weapons, anything you can think of.

How It Works

Switch to the 3D Modeler tab in the bottom panel. Click "+ New Model" to start, or pick one of the presets (House, Tree, Vehicle, Sword, Gem) to get a head start. Then use the primitive buttons (Box, Sphere, Cylinder, Cone, Torus, Plane) to add shapes. Each shape can be moved, rotated, scaled, and colored independently.

When you're happy with your model, hit the purple Save as Prefab button to turn it into a reusable prefab. Then you can spawn copies of it all over your game.

Modeler in Code

JavaScript
import { ModelBuilder } from './engine/ModelBuilder.js';

const modeler = new ModelBuilder(engine);

// build a custom house
const house = modeler.create('My House');
modeler.addPrimitive(house, 'box', {
  size: { x: 2, y: 1.5, z: 2 },
  color: '#cc8844',
  position: { x: 0, y: 0.75, z: 0 }
});
modeler.addPrimitive(house, 'cone', {
  radius: 1.5, height: 1.2, segments: 4,
  color: '#dd4444',
  position: { x: 0, y: 2.1, z: 0 }
});

// or use a preset to get started fast
const tree = modeler.presetTree('My Tree');
const sword = modeler.presetSword('Cool Sword');

// show it in the scene
modeler.showInScene(house);

// turn it into a prefab for your game
const prefabData = modeler.toPrefab(house, 'House');

// or spawn it directly into the scene
modeler.spawnModel(house, { x: 5, y: 0, z: 3 });
💡

Presets are a great starting point. Try presetHouse(), presetTree(), presetVehicle(), presetSword(), or presetGem(), then customize the parts to make them your own.

📌

Models can be exported as .novamodel files and shared between projects. You can also import them back into the modeler to keep editing.

Scripting

NovaScript Language

NovaScript is NovaStar's custom scripting language — a readable mix of C-style syntax designed for game entities. Use it to define behaviors, handle collisions, and control game logic.

NovaScript
entity Player {
  var speed: float = 5.0;
  var jumpForce: float = 12.0;
  var stars: int = 0;

  fn onUpdate(dt: float) {
    let move = Input.getMovement();
    this.velocity.x += move.x * speed * dt;

    if (Input.isActionJustPressed("jump") && this.isGrounded) {
      this.velocity.y = jumpForce;
      Audio.play("jump");
      Particles.jumpEffect(this.position);
    }
  }

  fn onCollision(other: Entity) {
    if (other.tag == "coin") {
      destroy(other);
      Audio.play("collect");
      Particles.collectEffect(other.position);
      stars += 1;
    }

    if (other.tag == "enemy") {
      Audio.play("hurt");
      this.velocity.y = 8.0; // bounce off
    }
  }
}

You can test NovaScript commands live in the Console panel at the bottom of the editor.
